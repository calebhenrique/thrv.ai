## Padrões de Desenvolvimento

### Nomenclatura de Branches
- `feature/nome-feature` - Para novas funcionalidades
- `fix/nome-bug` - Para correções de bugs
- `docs/nome-doc` - Para documentação
- `improve/nome-melhoria` - Para melhorias

### Commits
- Usar mensagens imperativas ("Add", "Fix", "Improve", "Refactor")
- Ser descritivo e conciso
- Referenciar issues quando aplicável

### Markdown
- Usar heading levels apropriados (#, ##, ###)
- Code blocks com syntax highlighting
- Listas bem formatadas
- Links quando relevante

## Código de Conduta
